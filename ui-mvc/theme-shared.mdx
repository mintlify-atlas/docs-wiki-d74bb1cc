`Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared` is the **base** every concrete
ABP MVC theme (Basic, LeptonX Lite, commercial themes) builds on. It is
not a theme itself — it doesn't ship layouts you would render directly —
but it owns the cross‑cutting pieces themes need to agree on: the global
script/style bundle names, the error page convention, the
`IToolbarManager`/`IPageToolbarManager` infrastructure that powers a
theme's main toolbar and per‑page action buttons, the shared view
components (application path, page search box, page toolbar), and the
embedded `_ViewStart.cshtml` files that route every page through the
current theme's layout.

## Module entry point

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/AbpAspNetCoreMvcUiThemeSharedModule.cs"
[DependsOn(
    typeof(AbpAspNetCoreMvcUiBootstrapModule),
    typeof(AbpAspNetCoreMvcUiPackagesModule),
    typeof(AbpAspNetCoreMvcUiWidgetsModule)
)]
public class AbpAspNetCoreMvcUiThemeSharedModule : AbpModule
{
    public override void PreConfigureServices(ServiceConfigurationContext context)
    {
        PreConfigure<IMvcBuilder>(mvcBuilder =>
        {
            mvcBuilder.AddApplicationPartIfNotExists(typeof(AbpAspNetCoreMvcUiThemeSharedModule).Assembly);
        });
    }

    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpVirtualFileSystemOptions>(options =>
        {
            options.FileSets.AddEmbedded<AbpAspNetCoreMvcUiThemeSharedModule>(
                "Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared");
        });

        Configure<AbpBundlingOptions>(options =>
        {
            options.StyleBundles
                .Add(StandardBundles.Styles.Global,
                    bundle => { bundle.AddContributors(typeof(SharedThemeGlobalStyleContributor)); });

            options.ScriptBundles
                .Add(StandardBundles.Scripts.Global,
                    bundle => bundle.AddContributors(typeof(SharedThemeGlobalScriptContributor)));
        });
    }
}
```

Three things happen at startup:

1. The assembly is registered as an MVC application part so its embedded
   Razor views and view components are discoverable.
2. The assembly is mounted into the [Virtual File System](/vfs/overview)
   so themes (and applications) can override its files at any virtual
   path.
3. Two **global bundles** are declared — one CSS, one JS — using
   `SharedThemeGlobalStyleContributor` and `SharedThemeGlobalScriptContributor`
   from the [Packages](/ui-mvc/packages-tag-helpers) catalogue.

## StandardBundles

The bundle names are constants so themes and applications can both
reference them without hard‑coding strings:

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Bundling/StandardBundles.cs"
public class StandardBundles
{
    public static class Styles
    {
        public static string Global = "Global";
    }

    public static class Scripts
    {
        public static string Global = "Global";
    }
}
```

A theme layout therefore renders the global bundle as:

```cshtml
<abp-style-bundle  name="@StandardBundles.Styles.Global" />
@* ...page body... *@
<abp-script-bundle name="@StandardBundles.Scripts.Global" />
```

## Shared global contributors

`SharedThemeGlobalScriptContributor` is the meta-contributor that pulls in
every JavaScript library a typical ABP MVC application expects: jQuery,
Bootstrap, jQuery validation, Select2, DataTables (Bootstrap 5),
SweetAlert2, Toastr, Luxon, the Bootstrap datepicker family and a couple
of `abp.*` extension scripts.


Applications that need to add to the global bundle should not redefine
it — they should extend the contributor through
`AbpBundleContributorOptions.Extensions<...>()` as described in
[Bundling](/ui-mvc/bundling).

## Error page

The shared module ships a single MVC controller and matching view that
themes reuse out of the box. The controller resolves the right error view
through `AbpErrorPageOptions.ErrorViewUrls`, falling back to a default
view for unknown HTTP status codes.

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Controllers/ErrorController.cs"
public class ErrorController : AbpController
{
    public virtual async Task<IActionResult> Index(int httpStatusCode)
    {
        var exHandlerFeature = HttpContext.Features.Get<IExceptionHandlerFeature>();

        var exception = exHandlerFeature != null
            ? exHandlerFeature.Error
            : new Exception(Localizer["UnhandledException"]);

        await ExceptionNotifier.NotifyAsync(new ExceptionNotificationContext(exception));

        var errorInfo = ErrorInfoConverter.Convert(exception, options =>
        {
            options.SendExceptionsDetailsToClients = ExceptionHandlingOptions.SendExceptionsDetailsToClients;
            options.SendStackTraceToClients        = ExceptionHandlingOptions.SendStackTraceToClients;
        });

        if (httpStatusCode == 0)
        {
            httpStatusCode = (int)StatusCodeFinder.GetStatusCode(HttpContext, exception);
        }

        HttpContext.Response.StatusCode = httpStatusCode;

        var page = GetErrorPageUrl(httpStatusCode);

        return View(page, new AbpErrorViewModel
        {
            ErrorInfo      = errorInfo,
            HttpStatusCode = httpStatusCode
        });
    }

    protected virtual string GetErrorPageUrl(int statusCode)
    {
        var page = AbpErrorPageOptions.ErrorViewUrls.GetOrDefault(statusCode.ToString());
        if (string.IsNullOrWhiteSpace(page))
            return "~/Views/Error/Default.cshtml";
        return page;
    }
}
```

`AbpErrorPageOptions.ErrorViewUrls` is the seam for overriding the view per
status code (e.g. supplying a branded `403.cshtml`):

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/AbpErrorPageOptions.cs"
public class AbpErrorPageOptions
{
    public readonly IDictionary<string, string> ErrorViewUrls;
    public AbpErrorPageOptions() { ErrorViewUrls = new Dictionary<string, string>(); }
}
```

An application opts into the error page convention with a single line:

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/AbpApplicationBuilderErrorPageExtensions.cs"
public static class AbpApplicationBuilderErrorPageExtensions
{
    public static IApplicationBuilder UseErrorPage(this IApplicationBuilder app)
    {
        return app
            .UseStatusCodePagesWithRedirects("~/Error?httpStatusCode={0}")
            .UseExceptionHandler("/Error");
    }
}
```

The default `Views/Error/Default.cshtml` lives in the embedded VFS and
renders an `AbpErrorViewModel` containing the converted `RemoteServiceErrorInfo`.

## Toolbar infrastructure

A theme's top toolbar (language switcher, user menu, tenant switch button)
is a composition of items contributed by modules. The shared module
defines the abstractions and the runtime; themes render a `Toolbar` into
HTML.

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Toolbars/AbpToolbarOptions.cs"
public class AbpToolbarOptions
{
    public List<IToolbarContributor> Contributors { get; }
    public AbpToolbarOptions() { Contributors = new List<IToolbarContributor>(); }
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Toolbars/StandardToolbars.cs"
public static class StandardToolbars
{
    public const string Main = "Main";
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Toolbars/IToolbarContributor.cs"
public interface IToolbarContributor
{
    Task ConfigureToolbarAsync(IToolbarConfigurationContext context);
}
```

`ToolbarManager` runs every contributor for the requested toolbar name and
filters items whose permissions don't hold. Permission filtering uses
ABP's [simple state checking](/authz/overview) infrastructure so contributors
can attach `RequirePermissions(...)` or arbitrary `ISimpleStateChecker`
rules to a `ToolbarItem`:

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/Toolbars/ToolbarManager.cs"
public class ToolbarManager : IToolbarManager, ITransientDependency
{
    public async Task<Toolbar> GetAsync(string name)
    {
        var toolbar = new Toolbar(name);

        using (var scope = ServiceProvider.CreateScope())
        using (RequirePermissionsSimpleBatchStateChecker<ToolbarItem>.Use(...))
        {
            var context = new ToolbarConfigurationContext(ThemeManager.CurrentTheme, toolbar, scope.ServiceProvider);
            foreach (var contributor in Options.Contributors)
                await contributor.ConfigureToolbarAsync(context);
            await CheckPermissionsAsync(scope.ServiceProvider, toolbar);
        }
        return toolbar;
    }
}
```

A real contributor looks like:

```csharp
public class MyToolbarContributor : IToolbarContributor
{
    public Task ConfigureToolbarAsync(IToolbarConfigurationContext context)
    {
        if (context.Toolbar.Name == StandardToolbars.Main)
        {
            context.Toolbar.Items.Add(new ToolbarItem(
                typeof(MyMenuViewComponent),
                requiredPermissionName: "MyApp.MyFeature"));
        }
        return Task.CompletedTask;
    }
}
```

## Page toolbars

`IPageToolbarManager` is the per‑page version: each Razor Page can have an
associated set of buttons (e.g. *New*, *Export*) populated through
`PageToolbarContributor`s registered under the page name.

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/PageToolbars/AbpPageToolbarOptions.cs"
public class AbpPageToolbarOptions
{
    public PageToolbarDictionary Toolbars { get; }

    public void Configure<TPage>(Action<PageToolbar> configureAction)
        => Configure(typeof(TPage).FullName!, configureAction);

    public void Configure(string pageName, Action<PageToolbar> configureAction)
    {
        var toolbar = Toolbars.GetOrAdd(pageName, () => new PageToolbar(pageName));
        configureAction(toolbar);
    }
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/PageToolbars/IPageToolbarManager.cs"
public interface IPageToolbarManager
{
    Task<PageToolbarItem[]> GetItemsAsync(string pageName);
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/PageToolbars/PageToolbarManager.cs"
public virtual async Task<PageToolbarItem[]> GetItemsAsync(string pageName)
{
    var toolbar = Options.Toolbars.GetOrDefault(pageName);
    if (toolbar == null || !toolbar.Contributors.Any())
        return Array.Empty<PageToolbarItem>();

    using (var scope = ServiceScopeFactory.CreateScope())
    {
        var context = new PageToolbarContributionContext(pageName, scope.ServiceProvider);
        foreach (var contributor in toolbar.Contributors)
            await contributor.ContributeAsync(context);
        return context.Items.OrderBy(i => i.Order).ToArray();
    }
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Theme.Shared/PageToolbars/IPageToolbarContributor.cs"
public interface IPageToolbarContributor
{
    Task ContributeAsync(PageToolbarContributionContext context);
}
```

A common contributor is `SimplePageToolbarContributor` which lets a module
declare items inline:

```csharp
Configure<AbpPageToolbarOptions>(options =>
{
    options.Configure<MyApp.Pages.Books.IndexModel>(toolbar =>
    {
        toolbar.AddButton(L["NewBook"], "btn btn-primary", icon: "fa fa-plus");
    });
});
```

Themes render the result by invoking `AbpPageToolbarViewComponent` near
the page title.

## View imports / view starts

The shared module also ships embedded `_ViewStart.cshtml` files that wire
every Razor view (`Views/_ViewStart.cshtml`), Razor Page
(`Pages/_ViewStart.cshtml`), Areas page
(`Areas/_ViewStart.cshtml`) and Account page
(`Pages/Account/_ViewStart.cshtml`) to use the layout returned by
`IThemeManager.CurrentTheme.GetLayout(...)`. That's why creating a new
Razor Page in an ABP MVC application doesn't require any layout setup at
all — the theme is chosen by configuration.
