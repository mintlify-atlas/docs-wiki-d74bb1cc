`Volo.Abp.AspNetCore.Mvc.UI.Packages` is the catalogue of `BundleContributor`
implementations that wrap the JavaScript and CSS libraries ABP applications
typically need. Every directory under
`framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/`
corresponds to one third‑party package — `Core` (abp.js / abp.css),
`JQuery`, `Bootstrap`, `DatatablesNetBs5`, `Select2`, `Moment`, `Luxon`,
`SweetAlert2`, `Toastr` and many more. Each contributor declares its
dependencies on other contributors and registers the right files from
`/libs/...`, including culture‑specific language packs delivered through
the `ConfigureDynamicResources` hook of [bundling abstractions](/ui-mvc/bundling-abstractions).

## Module entry point

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/AbpAspNetCoreMvcUiPackagesModule.cs"
[DependsOn(
    typeof(AbpAspNetCoreMvcUiBundlingAbstractionsModule),
    typeof(AbpLocalizationModule)
)]
public class AbpAspNetCoreMvcUiPackagesModule : AbpModule
{
    public override void ConfigureServices(ServiceConfigurationContext context)
    {
        Configure<AbpLocalizationOptions>(options =>
        {
            // BootstrapDatepicker
            options.AddLanguagesMapOrUpdate(BootstrapDatepickerScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh-CN"),
                new NameValue("zh-Hant", "zh-TW"));

            options.AddLanguageFilesMapOrUpdate(BootstrapDatepickerScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh-CN"),
                new NameValue("zh-Hant", "zh-TW"));

            // moment
            options.AddLanguagesMapOrUpdate(MomentScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh-cn"),
                new NameValue("zh-Hant", "zh-tw"),
                new NameValue("de-DE", "de"));

            options.AddLanguageFilesMapOrUpdate(MomentScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh-cn"),
                new NameValue("zh-Hant", "zh-tw"),
                new NameValue("de-DE", "de"));

            // Timeago
            options.AddLanguageFilesMapOrUpdate(TimeagoScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh-CN"),
                new NameValue("zh-Hant", "zh-TW"));

            // JQueryValidation
            options.AddLanguageFilesMapOrUpdate(JQueryValidationScriptContributor.PackageName,
                new NameValue("zh-Hans", "zh"),
                new NameValue("zh-Hant", "zh_TW"));
        });
    }
}
```

The module's only job at startup is to populate **language code maps** in
`AbpLocalizationOptions` so each package's dynamic resource lookup knows
how to translate an ASP.NET culture name (e.g. `zh-Hans`) into the file
name a third‑party library actually uses on disk (e.g. `zh-CN`). The
contributors themselves are not registered as DI services — they are
plain `new`‑able classes added to a `BundleConfiguration` from a
consuming module.

<Info>
This package depends only on **Bundling.Abstractions**, not on Bundling.
That keeps it usable from class libraries that compose bundle definitions
without needing the runtime (typical for module class libraries that ship
contributor types only).
</Info>

## Contributor shape

Every contributor follows the same pattern:

1. Inherit `BundleContributor` (from
   [Bundling abstractions](/ui-mvc/bundling-abstractions)).
2. Declare `[DependsOn(typeof(OtherContributor))]` for prerequisites so
   the right ordering happens automatically inside the
   `BundleContributorCollection`.
3. Override `ConfigureBundle(BundleConfigurationContext context)` to add
   the package's main JS/CSS files.
4. (Optional) Override `ConfigureDynamicResources` to add per‑culture
   files, gated by `context.FileProvider.GetFileInfo(...).Exists`.
5. (Optional) Expose `public const string PackageName = "..."` so the
   module can register a language map without referencing the file path.

### Core contributors

The `Core` directory contains the foundation files used by every other
contributor — `abp.js` and `abp.css` — and is therefore a dependency of
almost everything else.

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/Core/CoreScriptContributor.cs"
[DependsOn(typeof(UtilsScriptContributor))]
public class CoreScriptContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/abp/core/abp.js");
    }
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/Core/CoreStyleContributor.cs"
public class CoreStyleContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/abp/core/abp.css");
    }
}
```

### jQuery → Bootstrap chain

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/JQuery/JQueryScriptContributor.cs"
[DependsOn(typeof(CoreScriptContributor))]
public class JQueryScriptContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/jquery/jquery.js");
        context.Files.AddIfNotContains("/libs/abp/jquery/abp.jquery.js");
    }
}
```

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/Bootstrap/BootstrapScriptContributor.cs"
[DependsOn(typeof(JQueryScriptContributor))]
public class BootstrapScriptContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/bootstrap/js/bootstrap.bundle.js");
    }
}
```

Bootstrap's style contributor flips between LTR and RTL based on the
current culture — note that the file decision is taken at bundle time, so
RTL and LTR end up in separate cached bundles (different file lists →
different MD5 → different physical file on disk).

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/Bootstrap/BootstrapStyleContributor.cs"
public class BootstrapStyleContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        if (CultureHelper.IsRtl)
        {
            context.Files.AddIfNotContains("/libs/bootstrap/css/bootstrap.rtl.css");
        }
        else
        {
            context.Files.AddIfNotContains("/libs/bootstrap/css/bootstrap.css");
        }
    }
}
```

### Dynamic resources — culture-aware locales

A contributor with a language pack uses `ConfigureDynamicResources` rather
than `ConfigureBundle` so the per‑culture file is **not** baked into the
shared bundle. The bundle manager treats dynamic resources separately and
serves them outside the cached merged file.

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/Select2/Select2ScriptContributor.cs"
[DependsOn(typeof(CoreScriptContributor))]
public class Select2ScriptContributor : BundleContributor
{
    public const string PackageName = "select2";

    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/select2/js/select2.min.js");
    }

    public override void ConfigureDynamicResources(BundleConfigurationContext context)
    {
        var fileName = context.LazyServiceProvider
            .LazyGetRequiredService<IOptions<AbpLocalizationOptions>>()
            .Value.GetCurrentUICultureLanguageFilesMap(PackageName);

        var filePath = $"/libs/select2/js/i18n/{fileName}.js";
        if (context.FileProvider.GetFileInfo(filePath).Exists)
        {
            context.Files.AddIfNotContains(filePath);
        }
    }
}
```

The `Moment`, `Timeago`, `BootstrapDatepicker`, `BootstrapDaterangepicker`,
`JQueryValidation` and `Luxon` contributors follow the same shape; the
module's `AddLanguageFilesMapOrUpdate` calls are what map the culture
code to the local file name.

### Layered chains — DataTables + Bootstrap 5

DataTables uses three contributor layers — base library → Bootstrap 5
wrapper → application:

```csharp title="framework/src/Volo.Abp.AspNetCore.Mvc.UI.Packages/Volo/Abp/AspNetCore/Mvc/UI/Packages/DatatablesNetBs5/DatatablesNetBs5ScriptContributor.cs"
[DependsOn(typeof(DatatablesNetScriptContributor))]
[DependsOn(typeof(BootstrapScriptContributor))]
public class DatatablesNetBs5ScriptContributor : BundleContributor
{
    public override void ConfigureBundle(BundleConfigurationContext context)
    {
        context.Files.AddIfNotContains("/libs/datatables.net-bs5/js/dataTables.bootstrap5.js");
    }
}
```

`DatatablesNetBs4ScriptContributor` exists alongside for Bootstrap 4
applications.

## Composing contributors into a bundle

The two recommended patterns mirror the rest of the bundling stack:

```csharp
Configure<AbpBundlingOptions>(options =>
{
    options.ScriptBundles.Add("MyApp.Library", bundle =>
    {
        bundle.AddContributors(
            typeof(SignalRBrowserScriptContributor),
            typeof(ChartjsScriptContributor),
            typeof(SweetAlert2.Sweetalert2ScriptContributor));
    });

    options.StyleBundles.Add("MyApp.Library", bundle =>
    {
        bundle.AddContributors(
            typeof(FontAwesomeStyleContributor),
            typeof(TuiEditorStyleContributor));
    });
});
```

Or, to extend a contributor that already runs (e.g. the global one used by
[Theme.Shared](/ui-mvc/theme-shared)):

```csharp
Configure<AbpBundleContributorOptions>(options =>
{
    options.Extensions<SharedThemeGlobalScriptContributor>()
           .Add<MyAppExtraScriptContributor>();
});
```

## abp-* tag helpers in this package

Unlike [Bundling](/ui-mvc/bundling) or
[Bootstrap](/ui-mvc/bootstrap), this package **does not declare any tag
helpers**. Its sole responsibility is to expose `BundleContributor`
implementations. The `abp-script`, `abp-style`, `abp-script-bundle` and
`abp-style-bundle` tag helpers used to consume these contributors live in
the bundling assembly — see
[Bundling: tag helpers](/ui-mvc/bundling#tag-helpers).

## File inventory

The package layout is highly regular: every package gets its own folder
named after the library. Each folder contains one or two contributors
(`*ScriptContributor.cs`, `*StyleContributor.cs`) plus, when needed, an
options class (`JsTreeOptions.cs`). The module root is
`AbpAspNetCoreMvcUiPackagesModule.cs`.

| Folder | Files | Purpose |
| --- | --- | --- |
| `Core/` | `CoreScriptContributor.cs`, `CoreStyleContributor.cs` | `abp.js` / `abp.css` |
| `JQuery/`, `Bootstrap/`, ... | `*ScriptContributor.cs`, `*StyleContributor.cs` | Per‑library contributors |
| `JsTree/` | `JsTreeOptions.cs` (also) | Holds runtime options exposed via the contributor |
| Root | `AbpAspNetCoreMvcUiPackagesModule.cs` | Language map registrations |
