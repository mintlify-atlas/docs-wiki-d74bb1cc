`@abp/ng.account` is the visible half of the ABP Angular account flow. It bundles the seven authentication / profile screens, a routing module that maps `/account/*` URLs to them through `ReplaceableRouteContainerComponent`, two guards (`authenticationFlowGuard` and `AccountExtensionsGuard`), and one resolver that drives the extensible personal-settings form. The package depends on `@abp/ng.account.core` for proxies and visibility logic, and on `@abp/ng.theme.shared` for toasts and the password validator helpers.

Use this page as the per-file reference: every component, guard, token and route handler lives under `src/lib/`, and each is shown below with its real signature.

<Info>
  Published as **`@abp/ng.account`** from `npm/ng-packs/packages/account`. Pulls in `@abp/ng.account.core ~8.0.x` and `@abp/ng.theme.shared` as runtime dependencies, plus `@ng-bootstrap/ng-bootstrap`, `@ngx-validate/core`, and `@abp/ng.components/extensible`.
</Info>

## AccountModule

```ts packages/account/src/lib/account.module.ts
const declarations = [
  LoginComponent,
  RegisterComponent,
  ChangePasswordComponent,
  ManageProfileComponent,
  PersonalSettingsComponent,
  ForgotPasswordComponent,
  ResetPasswordComponent,
  PersonalSettingsHalfRowComponent,
];

@NgModule({
  declarations: [...declarations],
  imports: [
    CoreModule,
    AccountRoutingModule,
    ThemeSharedModule,
    NgbDropdownModule,
    NgxValidateCoreModule,
    ExtensibleModule,
  ],
  exports: [...declarations],
})
export class AccountModule {
  static forChild(options = {} as AccountConfigOptions): ModuleWithProviders<AccountModule> {
    return {
      ngModule: AccountModule,
      providers: [
        AuthenticationFlowGuard,
        { provide: ACCOUNT_CONFIG_OPTIONS, useValue: options },
        {
          provide: 'ACCOUNT_OPTIONS',
          useFactory: accountConfigOptionsFactory,
          deps: [ACCOUNT_CONFIG_OPTIONS],
        },
        {
          provide: RE_LOGIN_CONFIRMATION_TOKEN,
          useValue: options.isPersonalSettingsChangedConfirmationActive ?? true,
        },
        {
          provide: ACCOUNT_EDIT_FORM_PROP_CONTRIBUTORS,
          useValue: options.editFormPropContributors,
        },
        AccountExtensionsGuard,
      ],
    };
  }

  static forLazy(options = {} as AccountConfigOptions): NgModuleFactory<AccountModule> {
    return new LazyModuleFactory(AccountModule.forChild(options));
  }
}
```

Use `forChild` from a lazy-loaded feature module; `forLazy` is what `loadChildren: () => import(...).then(m => m.AccountModule.forLazy())` returns when registered through `RoutesService` in `app-routing.module.ts`.

### AccountConfigOptions

```ts packages/account/src/lib/models/config-options.ts
export interface AccountConfigOptions {
  redirectUrl?: string;
  isPersonalSettingsChangedConfirmationActive?: boolean;
  editFormPropContributors?: AccountEditFormPropContributors;
}
export type AccountEditFormPropContributors = Partial<{
  [eAccountComponents.PersonalSettings]: EditFormPropContributorCallback<UpdateProfileDto>[];
}>;
```

- `redirectUrl` becomes the post-login navigation target through `getRedirectUrl(injector)` in `utils/auth-utils.ts`.
- `isPersonalSettingsChangedConfirmationActive` (default `true`) forces a re-login confirmation when personal info changes.
- `editFormPropContributors` injects extra form props into the Manage Profile screen via `@abp/ng.components/extensible`.

## Routing

```ts packages/account/src/lib/account-routing.module.ts
const canActivate = [authenticationFlowGuard];

const routes: Routes = [
  { path: '', pathMatch: 'full', redirectTo: 'login' },
  {
    path: '',
    component: RouterOutletComponent,
    children: [
      {
        path: 'login',
        component: ReplaceableRouteContainerComponent,
        canActivate,
        data: {
          replaceableComponent: {
            key: eAccountComponents.Login,
            defaultComponent: LoginComponent,
          } as ReplaceableComponents.RouteData<LoginComponent>,
        },
      },
      {
        path: 'register',
        component: ReplaceableRouteContainerComponent,
        canActivate,
        data: {
          replaceableComponent: {
            key: eAccountComponents.Register,
            defaultComponent: RegisterComponent,
          } as ReplaceableComponents.RouteData<RegisterComponent>,
        },
      },
      {
        path: 'forgot-password',
        component: ReplaceableRouteContainerComponent,
        canActivate,
        data: {
          replaceableComponent: {
            key: eAccountComponents.ForgotPassword,
            defaultComponent: ForgotPasswordComponent,
          } as ReplaceableComponents.RouteData<ForgotPasswordComponent>,
        },
      },
      {
        path: 'reset-password',
        component: ReplaceableRouteContainerComponent,
        canActivate: [],
        data: {
          tenantBoxVisible: false,
          replaceableComponent: {
            key: eAccountComponents.ResetPassword,
            defaultComponent: ResetPasswordComponent,
          } as ReplaceableComponents.RouteData<ResetPasswordComponent>,
        },
      },
      {
        path: 'manage',
        component: ReplaceableRouteContainerComponent,
        canActivate: [authGuard],
        resolve: [accountExtensionsResolver],
        data: {
          replaceableComponent: {
            key: eAccountComponents.ManageProfile,
            defaultComponent: ManageProfileComponent,
          } as ReplaceableComponents.RouteData<ManageProfileComponent>,
        },
      },
    ],
  },
];
```

| Path | Guard | Replaceable key | Default component |
| --- | --- | --- | --- |
| `login` | `authenticationFlowGuard` | `Account.LoginComponent` | `LoginComponent` |
| `register` | `authenticationFlowGuard` | `Account.RegisterComponent` | `RegisterComponent` |
| `forgot-password` | `authenticationFlowGuard` | `Account.ForgotPasswordComponent` | `ForgotPasswordComponent` |
| `reset-password` | _none_ (token-driven) | `Account.ResetPasswordComponent` | `ResetPasswordComponent` |
| `manage` | `authGuard` + `accountExtensionsResolver` | `Account.ManageProfileComponent` | `ManageProfileComponent` |

The `data.tenantBoxVisible: false` on `reset-password` is the signal that `AuthWrapperService` (see [Account Core](/ng/account-core)) reads to hide the tenant switcher when the user arrived via an email link.

## Components

### LoginComponent

```ts packages/account/src/lib/components/login/login.component.ts
@Component({ selector: 'abp-login', templateUrl: './login.component.html' })
export class LoginComponent implements OnInit {
  form!: UntypedFormGroup;
  inProgress?: boolean;
  isSelfRegistrationEnabled = true;
  authWrapperKey = eAccountComponents.AuthWrapper;

  constructor(
    protected injector: Injector,
    protected fb: UntypedFormBuilder,
    protected toasterService: ToasterService,
    protected authService: AuthService,
    protected configState: ConfigStateService,
  ) {}

  ngOnInit() { this.init(); this.buildForm(); }

  protected init() {
    this.isSelfRegistrationEnabled =
      ((this.configState.getSetting('Abp.Account.IsSelfRegistrationEnabled') as string) || '')
        .toLowerCase() !== 'false';
  }

  protected buildForm() {
    this.form = this.fb.group({
      username: ['', [required, maxLength(255)]],
      password: ['', [required, maxLength(128)]],
      rememberMe: [false],
    });
  }
  // onSubmit() delegates to authService.login(...) and navigates with getRedirectUrl(injector)
}
```

The submit handler calls `accountService.register({ userName, emailAddress, password, appName })` then chains into `authService.login(...)` via `switchMap`.

### ForgotPasswordComponent / ResetPasswordComponent

Both use `AccountService` from `@abp/ng.account.core/proxy`:

- **Forgot** calls `sendPasswordResetCode({ email, appName, returnUrl })` and shows a toast confirmation.
- **Reset** reads `userId` and `resetToken` from the query string, then calls `resetPassword({ userId, resetToken, password })`. It uses `getPasswordValidators(injector)` from `@abp/ng.theme.shared` so password policy matches what the server enforces.

### ManageProfileComponent + PersonalSettings

`ManageProfileComponent` is a tabbed container; each tab is one of the three sub-components:

| Tab | Component | Selector |
| --- | --- | --- |
| Personal info | `PersonalSettingsComponent` | `<abp-personal-settings>` |
| Half-width column | `PersonalSettingsHalfRowComponent` | `<abp-personal-settings-half-row>` |
| Change password | `ChangePasswordComponent` | `<abp-change-password>` |

`PersonalSettingsComponent` calls `ProfileService.update(...)` and, if `RE_LOGIN_CONFIRMATION_TOKEN` is `true`, runs `confirmationService.warn(...)` followed by `authService.logout()` so the user re-authenticates with the new identity. `ManageProfileStateService` holds the live `ProfileDto`:

```ts packages/account/src/lib/services/manage-profile.state.service.ts
@Injectable({ providedIn: 'root' })
export class ManageProfileStateService {
  private readonly store = new InternalStore({} as ManageProfileState);

  get createOnUpdateStream() { return this.store.sliceUpdate; }
  getProfile$(): Observable<ProfileDto> { return this.store.sliceState(state => state.profile); }
  getProfile(): ProfileDto              { return this.store.state.profile; }
  setProfile(profile: ProfileDto)       { this.store.patch({ profile }); }
}
```

`ChangePasswordComponent` calls `ProfileService.changePassword({ currentPassword, newPassword })` and re-uses the same `getPasswordValidators(injector)` validators.

## Guards

### authenticationFlowGuard

Blocks the `login`, `register`, and `forgot-password` routes when the application is running in **external auth** mode (e.g. PKCE redirect against OpenIddict / IdentityServer) â€” those screens belong to the server in that mode and we navigate away.

```ts packages/account/src/lib/guards/authentication-flow.guard.ts
export const authenticationFlowGuard: CanActivateFn = () => {
  const authService = inject(AuthService);
  if (authService.isInternalAuth) return true;
  authService.navigateToLogin();
  return false;
};
```

The class-based `AuthenticationFlowGuard` is kept around as `@deprecated` for backwards compatibility.

### AccountExtensionsGuard

Initialises the extensible-form prop contributors before any account route activates, so the `<abp-extensible-form>` inside Personal Settings already knows about merged `ACCOUNT_EDIT_FORM_PROP_CONTRIBUTORS`. Wired through `accountExtensionsResolver` on the `manage` route.

## Wiring it up

<Steps>
  <Step title="Add menu entries (eager)">
    ```ts app.module.ts
    imports: [
      AccountConfigModule.forRoot(),
      // ...
    ]
    ```
    See [Account Core](/ng/account-core) for what `AccountConfigModule` registers.
  </Step>
  <Step title="Lazy-load the UI">
    ```ts app-routing.module.ts
    {
      path: 'account',
      loadChildren: () =>
        import('@abp/ng.account').then(m => m.AccountModule.forLazy({
          redirectUrl: '/',
        })),
    }
    ```
  </Step>
  <Step title="Override a screen (optional)">
    ```ts
    replaceableComponents.add({
      key: eAccountComponents.Login,
      component: MyBrandedLoginComponent,
    });
    ```
  </Step>
  <Step title="Extend Personal Settings (optional)">
    Provide an `editFormPropContributors` map on `AccountModule.forLazy({ ... })` to add custom fields backed by `UpdateProfileDto`.
  </Step>
</Steps>
