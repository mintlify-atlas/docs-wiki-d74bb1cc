`@abp/ng.account.core` is the headless layer underneath the visible login / register / manage-profile screens. It carries everything that is **not** a component template: the typed HTTP proxies for `/api/account/*`, the lightweight services that decide whether the tenant switcher and local login buttons are visible, and an optional `config` subpath (`@abp/ng.account/config`) that registers menu entries with `RoutesService`. UI packages such as `@abp/ng.account` and `@abp/ng.theme.basic` import from here, never from `@abp/ng.account` directly, so that custom theme authors can build their own login pages while reusing the same backend bindings.

This page documents the published surface of the npm package, including its `proxy` secondary entrypoint and the `account/config` companion module that ships alongside the UI package.

<Info>
  The package is published as **`@abp/ng.account.core`** from `npm/ng-packs/packages/account-core`. Generated proxies live at the secondary entrypoint **`@abp/ng.account.core/proxy`**. Menu / route contribution is in the sibling **`@abp/ng.account/config`** entrypoint (under `packages/account/config`).
</Info>

## Public API surface

The barrel re-exports just two files — everything else is consumed via the `proxy` subpath:

```ts packages/account-core/src/public-api.ts
/*
 * Public API Surface of account-core
 */

export * from './lib/auth-wrapper.service';
export * from './lib/tenant-box.service';
```

```ts packages/account-core/proxy/src/public-api.ts

```

```ts packages/account-core/proxy/src/lib/index.ts
export * from './proxy/account';
export * from './proxy/identity';

import * as Identity from './identity';

```

Consumers can therefore use either flat imports (`import { ProfileService } from '@abp/ng.account.core/proxy'`) or the namespaced form (`Account.ProfileService`).

## AuthWrapperService — visibility logic for auth pages

`AuthWrapperService` is the small piece of state shared between the **AuthWrapper** layout component and the login / register / forgot-password screens. It answers three questions per render:

1. Is multi-tenancy enabled at all? (reads `configState.getDeep$('multiTenancy.isEnabled')`)
2. Is local login enabled? (reads the `Abp.Account.EnableLocalLogin` setting)
3. Should the tenant-switch box show on the current route? (checks `route.snapshot.data.tenantBoxVisible`)

```ts packages/account-core/src/lib/auth-wrapper.service.ts
@Injectable()
export class AuthWrapperService {
  isMultiTenancyEnabled$ = this.configState.getDeep$('multiTenancy.isEnabled');

  get enableLocalLogin$(): Observable<boolean> {
    return this.configState
      .getSetting$('Abp.Account.EnableLocalLogin')
      .pipe(map(value => value?.toLowerCase() !== 'false'));
  }

  tenantBoxKey = 'Account.TenantBoxComponent';
  route: ActivatedRoute;

  get isTenantBoxVisibleForCurrentRoute() {
    return this.getMostInnerChild().data.tenantBoxVisible ?? true;
  }

  get isTenantBoxVisible() {
    return this.isTenantBoxVisibleForCurrentRoute && this.multiTenancy.isTenantBoxVisible;
  }

  constructor(
    public readonly multiTenancy: MultiTenancyService,
    private configState: ConfigStateService,
    injector: Injector,
  ) {
    this.route = injector.get(ActivatedRoute);
  }

  private getMostInnerChild() {
    let child = this.route.snapshot;
    let depth = 0;
    const depthLimit = 10;
    while (child.firstChild && depth < depthLimit) {
      child = child.firstChild;
      depth++;
    }
    return child;
  }
}
```

| Member | Type | Source |
| --- | --- | --- |
| `isMultiTenancyEnabled$` | `Observable<boolean>` | `multiTenancy.isEnabled` deep field of app config |
| `enableLocalLogin$` | `Observable<boolean>` | Setting `Abp.Account.EnableLocalLogin` |
| `isTenantBoxVisibleForCurrentRoute` | `boolean` | Deepest child route's `data.tenantBoxVisible` |
| `isTenantBoxVisible` | `boolean` | Combined with `MultiTenancyService.isTenantBoxVisible` |
| `tenantBoxKey` | `string` | Replaceable-component key for theme overrides |

The `AccountRoutingModule` in `@abp/ng.account` sets `data: { tenantBoxVisible: false }` on the `reset-password` route — that data hop is exactly what `getMostInnerChild` walks down to.

## TenantBoxService — switch-tenant modal logic

`TenantBoxService` powers the "Tenant: …" badge on the login page. It looks up a tenant by name, persists it through `SessionStateService`, and refreshes the application state so subsequent requests carry the new `__tenant` header.

```ts packages/account-core/src/lib/tenant-box.service.ts
@Injectable()
export class TenantBoxService {
  currentTenant$ = this.sessionState.getTenant$();

  name?: string;
  isModalVisible!: boolean;
  modalBusy!: boolean;

  constructor(
    private toasterService: ToasterService,
    private tenantService: AbpTenantService,
    private sessionState: SessionStateService,
    private configState: ConfigStateService,
  ) {}

  onSwitch() {
    const tenant = this.sessionState.getTenant();
    this.name = tenant?.name || '';
    this.isModalVisible = true;
  }

  save() {
    if (!this.name) {
      this.setTenant(null);
      this.isModalVisible = false;
      return;
    }

    this.modalBusy = true;
    this.tenantService
      .findTenantByName(this.name)
      .pipe(finalize(() => (this.modalBusy = false)))
      .subscribe(({ success, tenantId: id, ...tenant }) => {
        if (!success) { this.showError(); return; }
        this.setTenant({ ...tenant, id, isAvailable: true });
        this.isModalVisible = false;
      });
  }

  private setTenant(tenant: CurrentTenantDto | null) {
    this.sessionState.setTenant(tenant);
    this.configState.refreshAppState();
  }

  private showError() {
    this.toasterService.error('AbpUiMultiTenancy::GivenTenantIsNotAvailable', 'AbpUi::Error', {
      messageLocalizationParams: [this.name || ''],
    });
  }
}
```

`AbpTenantService` is the core proxy for `/api/abp/multi-tenancy/tenants/by-name/{name}`. `refreshAppState()` re-fetches `/api/abp/application-configuration` so permissions, languages and current-tenant flow through `ConfigStateService` to the rest of the SPA.

## Proxies — `@abp/ng.account.core/proxy`

### Account (registration / forgotten password)

```ts packages/account-core/proxy/src/lib/proxy/account/account.service.ts
@Injectable({ providedIn: 'root' })
export class AccountService {
  apiName = 'AbpAccount';

  register = (input: RegisterDto) =>
    this.restService.request<any, IdentityUserDto>(
      { method: 'POST', url: '/api/account/register', body: input },
      { apiName: this.apiName },
    );

  resetPassword = (input: ResetPasswordDto) =>
    this.restService.request<any, void>(
      { method: 'POST', url: '/api/account/reset-password', body: input },
      { apiName: this.apiName },
    );

  sendPasswordResetCode = (input: SendPasswordResetCodeDto) =>
    this.restService.request<any, void>(
      { method: 'POST', url: '/api/account/send-password-reset-code', body: input },
      { apiName: this.apiName },
    );

  constructor(private restService: RestService) {}
}
```

### Profile (current-user CRUD)

```ts packages/account-core/proxy/src/lib/proxy/account/profile.service.ts
export class ProfileService {

      { apiName: this.apiName },
    );

  constructor(private restService: RestService) {}
}
```

### MVC login controller — `Account` namespace

The MVC area controller is exposed under `Account.AccountService` (separate from the application service `AccountService` above):

```ts packages/account-core/proxy/src/lib/proxy/account/web/areas/account/controllers/account.service.ts
@Injectable({ providedIn: 'root' })
export class AccountService {
  apiName = 'AbpAccount';

  checkPasswordByLogin = (login: UserLoginInfo) =>
    this.restService.request<any, AbpLoginResult>(
      { method: 'POST', url: '/api/account/check-password', body: login },
      { apiName: this.apiName },
    );

  loginByLogin = (login: UserLoginInfo) =>
    this.restService.request<any, AbpLoginResult>(
      { method: 'POST', url: '/api/account/login', body: login },
      { apiName: this.apiName },
    );

  logout = () =>
    this.restService.request<any, void>(
      { method: 'GET', url: '/api/account/logout' },
      { apiName: this.apiName },
    );

  constructor(private restService: RestService) {}
}
```

The `AbpLoginResult` payload is typed by `LoginResultType` (Success, InvalidUserNameOrPassword, NotAllowed, LockedOut, RequiresTwoFactor) — see `login-result-type.enum.ts`.

## `@abp/ng.account/config` — menu registration

The `account/config` entrypoint contributes the side-menu nodes that the `@abp/ng.theme.shared` chrome reads. It is imported once in `app.module.ts`:

```ts apps/dev/src/app/app.module.ts
imports: [
  AccountConfigModule.forRoot(),
  // ...
]
```

```ts packages/account/config/src/account-config.module.ts
@NgModule()
export class AccountConfigModule {
  static forRoot(): ModuleWithProviders<AccountConfigModule> {
    return {
      ngModule: AccountConfigModule,
      providers: [
        ACCOUNT_ROUTE_PROVIDERS,
        {
          provide: NAVIGATE_TO_MANAGE_PROFILE,
          useFactory: navigateToManageProfileFactory,
          deps: [Injector],
        },
      ],
    };
  }
}
```

`NAVIGATE_TO_MANAGE_PROFILE` is the injection token consumed by the `currentUser` toolbar dropdown in `@abp/ng.theme.shared`. The factory pushes the user to `/account/manage`, which keeps the toolbar decoupled from `@abp/ng.account`.

### Route contributions

```ts packages/account/config/src/providers/route.provider.ts
export function configureRoutes(routes: RoutesService) {
  return () => {
    routes.add([
      {
        path: undefined,
        name: eAccountRouteNames.Account,
        invisible: true,
        layout: eLayoutType.account,
        order: 1,
      },
      { path: '/account/login',           name: eAccountRouteNames.Login,         parentName: eAccountRouteNames.Account, order: 1 },
      { path: '/account/register',        name: eAccountRouteNames.Register,      parentName: eAccountRouteNames.Account, order: 2 },
      {
        path: '/account/manage',
        name: eAccountRouteNames.ManageProfile,
        parentName: eAccountRouteNames.Account,
        layout: eLayoutType.application,  // shows under the user menu, not the auth layout
        order: 3,
      },
      { path: '/account/forgot-password', name: eAccountRouteNames.ForgotPassword, parentName: eAccountRouteNames.Account, invisible: true },
      { path: '/account/reset-password',  name: eAccountRouteNames.ResetPassword,  parentName: eAccountRouteNames.Account, invisible: true },
    ]);
  };
}
```

## Configuration recipes

<CardGroup cols={2}>
  <Card title="Disable self-registration link" icon="user-slash">
    Set `Abp.Account.IsSelfRegistrationEnabled = "false"` on the server — `LoginComponent` and `AuthWrapperService` will react automatically via `ConfigStateService.getSetting$`.
  </Card>
  <Card title="Hide tenant box on a route" icon="building-shield">
    Add `data: { tenantBoxVisible: false }` to your Angular route. `AuthWrapperService.isTenantBoxVisibleForCurrentRoute` walks `route.snapshot` to find it.
  </Card>
  <Card title="External auth only" icon="key-skeleton">
    Set `Abp.Account.EnableLocalLogin = "false"` — `enableLocalLogin$` flips to `false` and the local form hides.
  </Card>
  <Card title="Custom manage-profile target" icon="user-pen">
    Override the `NAVIGATE_TO_MANAGE_PROFILE` provider after `AccountConfigModule.forRoot()` to redirect the user-menu link elsewhere (e.g. an internal HR page).
  </Card>
</CardGroup>

## Working with the proxies

<Steps>
  <Step title="Inject the typed service">

  </Step>

    ```ts
    this.profile.get().subscribe(p => this.state.setProfile(p));
    ```

  </Step>

  </Step>
