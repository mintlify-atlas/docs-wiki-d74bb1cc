`@abp/ng.identity` is the Angular feature module behind the **Administration → Identity Management → Users / Roles** screens. It owns the two list / edit components, the routing module guarded by `authGuard` + `permissionGuard`, five extension-token surfaces for customising the grids and forms, and a separate `proxy` entrypoint that exposes typed services for `/api/identity/users`, `/api/identity/roles`, and `/api/identity/users/lookup`. A `config` entrypoint contributes the side-menu nodes and is loaded eagerly at app boot.

This page lists every notable file under `src/lib`, shows the real component class signatures, and documents the proxy services and extension hooks.

<Info>
  Published as **`@abp/ng.identity`** from `npm/ng-packs/packages/identity`. Proxies live at **`@abp/ng.identity/proxy`** (`packages/identity/proxy`). Menu / route registration lives at **`@abp/ng.identity/config`** (`packages/identity/config`).
</Info>

## IdentityModule

```ts packages/identity/src/lib/identity.module.ts
@NgModule({
  declarations: [RolesComponent, UsersComponent],
  exports:     [RolesComponent, UsersComponent],
  imports: [
    CoreModule,
    IdentityRoutingModule,
    NgbNavModule,
    ThemeSharedModule,
    ExtensibleModule,
    NgbDropdownModule,
    PermissionManagementModule,
    NgxValidateCoreModule,
    PageModule,
  ],
})
export class IdentityModule {
  static forChild(options: IdentityConfigOptions = {}): ModuleWithProviders<IdentityModule> {
    return {
      ngModule: IdentityModule,
      providers: [
        { provide: IDENTITY_ENTITY_ACTION_CONTRIBUTORS,     useValue: options.entityActionContributors },
        { provide: IDENTITY_TOOLBAR_ACTION_CONTRIBUTORS,    useValue: options.toolbarActionContributors },
        { provide: IDENTITY_ENTITY_PROP_CONTRIBUTORS,       useValue: options.entityPropContributors },
        { provide: IDENTITY_CREATE_FORM_PROP_CONTRIBUTORS,  useValue: options.createFormPropContributors },
        { provide: IDENTITY_EDIT_FORM_PROP_CONTRIBUTORS,    useValue: options.editFormPropContributors },
        IdentityExtensionsGuard,
      ],
    };
  }

  static forLazy(options: IdentityConfigOptions = {}): NgModuleFactory<IdentityModule> {
    return new LazyModuleFactory(IdentityModule.forChild(options));
  }
}
```

Note the implicit dependency on `PermissionManagementModule` — the user/role edit modals embed `<abp-permission-management>` to grant policies inline.

## Routing

```ts packages/identity/src/lib/identity-routing.module.ts
const routes: Routes = [
  { path: '', redirectTo: 'roles', pathMatch: 'full' },
  {
    path: '',
    component: RouterOutletComponent,
    canActivate: [authGuard, permissionGuard],
    resolve: [identityExtensionsResolver],
    children: [
      {
        path: 'roles',
        component: ReplaceableRouteContainerComponent,
        data: {
          requiredPolicy: 'AbpIdentity.Roles',
          replaceableComponent: {
            key: eIdentityComponents.Roles,
            defaultComponent: RolesComponent,
          } as ReplaceableComponents.RouteData<RolesComponent>,
        },
      },
      {
        path: 'users',
        component: ReplaceableRouteContainerComponent,
        data: {
          requiredPolicy: 'AbpIdentity.Users',
          replaceableComponent: {
            key: eIdentityComponents.Users,
            defaultComponent: UsersComponent,
          } as ReplaceableComponents.RouteData<UsersComponent>,
        },
      },
    ],
  },
];
```

- `permissionGuard` reads `data.requiredPolicy` and gates the route through `PermissionService.getGrantedPolicy$(...)`.
- `identityExtensionsResolver` ensures contributors have run before either grid renders so columns / actions are stable.


Key collaborators:

- `ListService<GetIdentityUsersInput>` — paged / sorted query loop (see [core](/ng/core)).
- `EXTENSIONS_IDENTIFIER` — tells the embedded `<abp-extensible-form>` and `<abp-extensible-table>` which contributor bag to read.
- `IdentityUserService` — proxy in `@abp/ng.identity/proxy`.
- `ePermissionManagementComponents.PermissionManagement` — used to open the per-user permission modal via `<abp-permission-management>`.

The defaults emitted by `defaults/*.ts` are merged with user contributions through `@abp/ng.components/extensible`:

| Token | Surface | Default constant |
| --- | --- | --- |
| `IDENTITY_ENTITY_ACTION_CONTRIBUTORS` | Row dropdown | `DEFAULT_IDENTITY_ENTITY_ACTIONS` |
| `IDENTITY_TOOLBAR_ACTION_CONTRIBUTORS` | Above-grid buttons | `DEFAULT_IDENTITY_TOOLBAR_ACTIONS` |
| `IDENTITY_ENTITY_PROP_CONTRIBUTORS` | Grid columns | `DEFAULT_IDENTITY_ENTITY_PROPS` |
| `IDENTITY_CREATE_FORM_PROP_CONTRIBUTORS` | Create modal | `DEFAULT_IDENTITY_CREATE_FORM_PROPS` |

```ts packages/identity/src/lib/tokens/extensions.token.ts
export const DEFAULT_IDENTITY_ENTITY_PROPS = {
  [eIdentityComponents.Roles]: DEFAULT_ROLES_ENTITY_PROPS,
```

`IdentityExtensionsGuard` (provided by `IdentityModule.forChild`) reads the tokens, runs each contributor through `mergeContributors(...)` and stores the merged sets in `ExtensionsService` — see [Components → extensible](/ng/components#extensible).

## Proxies — `@abp/ng.identity/proxy`

### IdentityUserService

```ts packages/identity/proxy/src/lib/proxy/identity/identity-user.service.ts
@Injectable({ providedIn: 'root' })
export class IdentityUserService {
  apiName = 'AbpIdentity';

  create = (input: IdentityUserCreateDto) =>
    this.restService.request<any, IdentityUserDto>(
      { method: 'POST', url: '/api/identity/users', body: input },
      { apiName: this.apiName });

  delete = (id: string) =>
    this.restService.request<any, void>(
      { method: 'DELETE', url: `/api/identity/users/${id}` },
      { apiName: this.apiName });

  findByEmail = (email: string) => /* GET /api/identity/users/by-email/{email} */;
  findByUsername = (userName: string) => /* GET /api/identity/users/by-username/{userName} */;
  get = (id: string) => /* GET /api/identity/users/{id} */;
  getAssignableRoles = () => /* GET /api/identity/users/assignable-roles */;

  getList = (input: GetIdentityUsersInput) =>
    this.restService.request<any, PagedResultDto<IdentityUserDto>>(
      {
        method: 'GET',
        url: '/api/identity/users',
        params: { filter: input.filter, sorting: input.sorting,
                  skipCount: input.skipCount, maxResultCount: input.maxResultCount },
      },
      { apiName: this.apiName });

  getRoles = (id: string) => /* GET /api/identity/users/{id}/roles */;
  update = (id: string, input: IdentityUserUpdateDto) => /* PUT /api/identity/users/{id} */;
  updateRoles = (id: string, input: IdentityUserUpdateRolesDto) =>
    /* PUT /api/identity/users/{id}/roles */;

  constructor(private restService: RestService) {}
}
```

### IdentityUserLookupService

Backs typeahead pickers in custom modules — exposes `findById`, `findByUserName`, `getCount(filter)`, and `search(input)` against `/api/identity/users/lookup/*`. Payloads use the lightweight `UserData` shape from `users/models.ts`.

## Adding a custom column to Users

<Steps>
  <Step title="Write a contributor">
    ```ts
    const usersEntityPropContributor: EntityPropContributorCallback<IdentityUserDto> = props => {
      props.addByIndex(
        new EntityProp<IdentityUserDto>({
          type: 'string',
          name: 'displayName',
          displayName: 'AbpIdentity::DisplayName',
          valueResolver: data => of(`${data.record.name} ${data.record.surname}`),
        }),
        3,
      );
    };
    ```
  </Step>
  <Step title="Register it on the lazy module">
    ```ts
    IdentityModule.forLazy({
      entityPropContributors: {
        [eIdentityComponents.Users]: [usersEntityPropContributor],
      },
    })
    ```
  </Step>
  <Step title="Reload the route">
    `IdentityExtensionsGuard` runs, contributors merge into the default set, and the grid renders the new column.
  </Step>
</Steps>
